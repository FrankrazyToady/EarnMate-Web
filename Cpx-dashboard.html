Postback Setup

The postback is your primary method to credit users. CPX Research calls your server URL every time a user completes or screens out of a survey.

⚠
Do not use the Statistics API as your main payment method. Use the postback function as primary integration — the API is only for occasional verification (e.g. once per hour).
Your Postback URL Structure
https://yourdomain.com/postback?transaction_id={transaction_id}&status={status}&amount={amount}&app_id={app_id}

Replace yourdomain.com/postback with your actual endpoint. CPX will send a GET request to this URL for every event.

Postback Parameters
Parameter Description Example
{transaction_id} Unique ID for this event. Use to prevent double-crediting. TX_abc123
{status} 1 = Complete   2 = Screen Out 1 2
{amount} Reward amount in your currency/points to credit the user. 0.50
{app_id} Your app ID from the CPX dashboard. 12345
{user_id} The user's ID in your system (you passed this when opening surveys). user_99
Validate Before Crediting
ℹ
Before crediting a user from a postback, always validate the transaction_id via the API to prevent fraud. Switch to the Validate TX tab to test this.
Example server-side handler (Node.js)
server.js 
app.get('/postback', async (req, res) => {
  // Extract postback params
  const { transaction_id, status, amount, user_id } = req.query;

  // 1. Validate transaction with CPX API
  const check = await fetch(`...api-check-transaction-id&transaction_id=${transaction_id}`);
  const data  = await check.json();
  if (!data.valid) return res.sendStatus(400);

  // 2. Prevent duplicate crediting (check your DB)
  if (await alreadyCredited(transaction_id)) return res.sendStatus(200);

  // 3. Credit the user based on status
  if (status === '1') await creditUser(user_id, amount);   // complete
  if (status === '2') await creditUser(user_id, amount * 0.1); // screen out

  res.sendStatus(200); // always return 200 to CPX
});
